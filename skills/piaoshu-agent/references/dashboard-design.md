| # | 设计决策 | Polsia 实现 | 我们怎么用 |

| Polsia 缺的 | 我们有的 |
|------------|---------|
| AI 身份归属不明 | AFC Chain 确权 → 分身的认知所有权可验证 |
| 没有去中心化身份层 | DID+SBT → 分身的数字人格锚定 |

## 设计原则

1. **透明度优先** — AI 的每一步推理、每一次搜索、每一个决策，用户都能看到。不黑盒。
2. **所有权可视** — 分身的训练数据来源、技能组成、知识图谱，可查看、可导出、可迁移。
3. **实时感** — 不是静态报表。流式更新，终端风格，仿生节奏。
4. **锚定身份** — 所有界面从 🔩 emoji + 一句话灵魂宣言开始。不迷失在数据里。
5. **终端美学** — 深色背景、等宽字体、`>` 前缀、ASCII 装饰。拒绝圆角卡片和渐变色。
6. **数据说话** — 指标带环比（WoW/MoM），引用带出处，决策带 Benchmark。

## 关键技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js + Tailwind | 独立应用，SSE 实时推送 |
| 实时数据 | Event Bus (Redis Streams → NATS) | swarm-scheduler.md定义 |
| 历史数据 | PostgreSQL + pgvector | 知识网格向量检索 |
| 链上数据 | AFC Indexer (The Graph) | 托管/结算/租赁事件 |
| 部署 | Vercel / Cloudflare Pages | 静态导出 + Edge Functions |
| 终端美学 | #0a0a0a + JetBrains Mono + #00ff00 | 深色Matrix风 |
